üìã Overview

The AFR_RDM (Africa Robust Decision Making) model is an essential tool that allows for the evaluation of multiple scenarios and uncertainties in energy optimization models. This guide provides step by step instructions for setting up and running RDM analyses.

6 Uncertainty_Table Sheet

6

Configure uncertainties

In the first sheet named Uncertainty_Table, you will define all uncertainties you want to evaluate in the experiment. Follow the column structure detailed in the next section.

üìä Column Configuration - Uncertainty_Table

Uncertainty table structure

The Uncertainty_Table sheet must be configured with the following columns (A through N):

Column Field Name Description Example
A X_Num Uncertainty number (ascending order) 1, 2, 3...
B X_Category Uncertainty category Fuel Costs, Capital Cost
C X_Plain_English_Description Brief description of the uncertainty Changes in imported energy carriers
D Involved_Scenarios Involved scenarios (separated by " ; ") Scenario1 ; Scenario2
E X_Mathematical_Type Mathematical type of the series:
‚Ä¢ Time_Series: Replicates original curve behavior
‚Ä¢ Constant: Constant curve
‚Ä¢ Logistic: Logistic curve
‚Ä¢ Linear: Linear curve		 Time_Series
F Explored_Parameter_of_X Parameter to explore Final_Value
G Initial_Year_of_Uncertainty Starting year for uncertainty variations 2025
H Min_Value Minimum variation value 0.5
I Max_Value Maximum variation value 1.5
J Exact_Parameters_Involved_in_Osemosys Exact OSeMOSYS parameters (separated by " ; ") VariableCost ; CapitalCost
K Involved_First_Sets_in_Osemosys Values of the first set used by the parameter (technologies or fuels) MIMBIO ; MINCOA
L Involved_Second_Sets_in_Osemosys Values of the second set (if applicable per Params_Sets_Vari sheet) BIO ; COA
M Involved_Third_Sets_in_Osemosys Values of the third set (if applicable per Params_Sets_Vari sheet) TS01 ; TS02
N Manipulation_of_Parameter Description of data manipulation according to X_Mathematical_Type Multiply final value by explored parameter and interpolate
üí° Important Notes about Sets
  • To determine which sets apply to a parameter, check the Params_Sets_Vari sheet
  • Each OSeMOSYS parameter column shows its required sets (excluding Region)
  • Parameters highlighted in brown do not apply for this functionality
  • If a parameter uses 2 sets: place first in column K, second in column L
  • If a parameter uses 3 sets: place first in column K, second in column L, third in column M
  • Always use exact format with spaces: Value1 ; Value2 ; Value3

üîß PRIM Module Configuration

Objective

Establish the steps and artifacts necessary to reproduce PRIM execution (Excel inputs, Python scripts, and YAML files).

Pipeline/Workflow Flow

Execution Order to Replicate PRIM:

1. Input Data Preparation

  • Complete src\workflow\4_PRIM\t3b_sdiscovery\Analysis_1\prim_structure.xlsx (Sequences and Outcomes sheets)
  • Fill src\workflow\4_PRIM\Population.xlsx file with demographic data
  • Configure src\workflow\4_PRIM\prim_files_creator_cntrl.xlsx with control parameters
  • Configure src\workflow\4_PRIM\t3b_sdiscovery\Units.xlsx with measurement units for drivers

2. YAML Configuration

  • Verify and adjust src\workflow\4_PRIM\PRIM_t3f2.yaml according to project needs

3. Python Script Execution (in order)

  • src\workflow\4_PRIM\t3b_sdiscovery\t3f1_prim_structure.py - Processes PRIM structure and generates prim_files_creator.pickle
  • src\workflow\4_PRIM\t3f2_prim_files_creator.py - Creates PRIM files using experiment data
  • src\workflow\4_PRIM\t3b_sdiscovery\t3f3_prim_manager.py - Executes PRIM analysis and generates results
  • src\workflow\4_PRIM\t3b_sdiscovery\t3f4_range_finder_mapping.py - Maps predominant driver ranges

4. Results Interpretation

  • Check the results in the file src\workflow\4_PRIM\t3b_sdiscovery\t3f4_predominant_ranges_a1_e1_Experiment.xlsx.
üìù Important Note
To complete the "Input Data Preparation", "YAML Configuration" and "Results Interpretation" steps, follow the "Guide PRIM Module Configuration".
