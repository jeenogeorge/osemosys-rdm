PRIM Module Configuration

Complete Guide for Parameter Setup and Template Configuration

Overview

Objective: Establish the steps and artifacts necessary to set parameters of the PRIM (Excel inputs and YAML files).

Pipeline of Set Parameters

Execution Order to Replicate PRIM:

  1. Input Data Preparation
    • Complete prim_structure.xlsx (Sequences and Outcomes sheets)
    • Fill Population.xlsx file with demographic data
    • Configure prim_files_creator_cntrl.xlsx with control parameters
    • Configure Units.xlsx with measurement units for drivers
  2. YAML Configuration
    • Verify and adjust PRIM_t3f2.yaml according to project needs

Required Inputs

  • t3b_sdiscovery/Analysis_1/prim_structure.xlsx (Sequences and Outcomes sheets)
  • Population.xlsx (historical/projected population)
  • prim_files_creator_cntrl.xlsx (execution control)
  • YAML configuration file: PRIM_t3f2.yaml
  • t3b_sdiscovery/Units.xlsx (unit definition per driver)
Important Notes:
  • Maintain column names exactly as they appear in the templates
  • Unused sheets in prim_structure.xlsx are: Indications, Qs_and_Notes, Command_Desc, Set_Matching_4_Param_Mult

Note: Only change the variables that have the comment beginning with “Write the.”

Final Reminders

Important Validations:

  1. Name consistency: Driver names in Units.xlsx must exactly match those generated by prim_structure.xlsx
  2. List format: Always use space + semicolon + space to separate elements (example: "value1 ; value2 ; value3")
  3. Complete periods: Ensure Population.xlsx covers all years defined in periods
  4. Valid scenarios: Scenarios in Driver_Involved_Scenarios must exist in experiments
  5. Data types: Verify types in dtype correspond with actual data

Generated Files:

  • prim_files_creator.pickle - Processed structure (Intermediate)
  • comp_pfd_*.pickle - Compiled data per analysis (Intermediate)
  • sd_ana_*_exp_*.csv - PRIM analysis results (Intermediate)
  • t3f4_predominant_ranges_*.xlsx - Mapped predominant ranges (Results)

Verification Order:

  1. Verify all Excel files are complete
  2. Confirm correct list format (space ; space)
  3. Execute t3f1 and verify pickle is generated correctly
  4. Review t3f2 logs to confirm experiment processing
  5. Validate t3f3 results in CSV file
  6. Confirm range mapping in t3f4

Results Interpretation

After executing the PRIM pipeline, the predominant ranges and key drivers are mapped in the output file located at:

src\workflow\4_PRIM\t3b_sdiscovery\t3f4_predominant_ranges_a1_e1_Experiment.xlsx

This file contains the identified uncertainty ranges and their relationship with outcomes. Below is the interpretation guide for each column in the results file:

Results Columns Interpretation

Column Interpretation
Outcome_Type Classification of the result type being analyzed: "Desirable" for desirable outcomes (costs and emissions to minimize) or "Risk" for risk zones to monitor.
Metric Type of metric used to evaluate the outcome: "Costs" (costs), "Emissions" (emissions), or "All" (multiple combined metrics).
Period Temporal range of the analysis expressed in years (e.g., "41-50", "21-30", "31-40"). Defines the evaluation horizon within the project lifecycle.
Driver Specific uncertainty factor being analyzed by PRIM (e.g., "Capital Costs ICE Transport", "Variable Costs Crops", "Fuel Demand"). Represents the parameter whose variability is being studied.
Driver_Type Classification of the driver type. Currently only "uncertainty" is available, indicating inherent variability of the parameter.
Min_Norm Minimum normalized value (between 0 and 1) of the uncertainty range identified by PRIM. Lower bound of the region where notable behavior is obtained.
Max_Norm Maximum normalized value (between 0 and 1) of the uncertainty range identified by PRIM. Upper bound of the region. Together with Min_Norm, they define the PRIM "box".
Case Identifier of the scenario or case being analyzed. Groups multiple drivers simultaneously, indicating critical combinations of uncertainties that generate interesting results.
Condition Robustness status of the result: "definitive" indicates that the result is conclusive and robust within the identified region.
Tiebreaker Additional criterion for resolving trade-offs: "none" (no special criterion) or "high/low average" (when there are conflicts between objectives, specifying whether to maximize or minimize).
Understanding the Results:
  • PRIM Boxes: The Min_Norm and Max_Norm values define "boxes" or regions in the uncertainty space where specific outcomes are most likely to occur.
  • Cases: Multiple drivers with the same Case number indicate that these uncertainties interact together to produce the observed outcome.
  • Normalized Values: All values are normalized between 0 and 1, making it easy to compare different uncertainty ranges regardless of their original units.
